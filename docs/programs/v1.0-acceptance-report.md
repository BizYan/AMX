## Tag Recommendation

Do not tag `v1.0.0` from this workspace evidence alone.

Tagging is reasonable only after:

1. real authenticated API smoke passes against the selected staging or production URL;
2. Docker Compose config passes in CI or release workflow;
3. migration upgrade is verified on a disposable or staging database;
4. owner confirms the selected release ref and rollback target.

If an intended tag already exists, do not move, recreate, or silently reuse it.
Select an Owner-approved version strategy and preserve exact SHA evidence for
the promoted tag.
