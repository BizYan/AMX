## Classification Rules

Capability status uses the Batch 0 goal categories:

- Real: implemented product surface with backend/domain support and no obvious reliance on deterministic mock fixtures for the core path.
- Mock-only: verified only through mocked API/test fixtures or deterministic frontend fixtures.
- Placeholder: code or tests explicitly allow placeholder/generated-placeholder states.
- Sandbox: uses sandbox/test/demo keys, data, provider names, or non-production setup.
- Adapter-stub: route/service shape exists, but production connector/provider behavior is not proven.
- Unknown: source inspection found a surface, but Batch 0 did not prove production behavior.

Mock/test fixtures are valuable regression assets but are not counted as production readiness evidence.
