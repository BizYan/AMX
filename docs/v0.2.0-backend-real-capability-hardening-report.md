# v0.2.0 后端真实能力与前端联调加固报告

本报告记录 `feature/v0.2.0-backend-real-capability-hardening` 分支引入并经二次审查收口后的事实状态。该分支目标是把若干前端已有入口从“仅有反馈或 Mock 覆盖”推进到真实后端 API、数据库表和端到端测试可验证的状态。

## 已交付内容

### 数据库迁移

- 新增 Alembic 迁移 `0003_add_v020_tables.py`，补齐以下 v0.2.0 相关物理表：
  - `project_settings`
  - `project_invitations`
  - `source_files`
  - `export_jobs`
  - `export_artifacts`
  - `templates`
  - `template_versions`
- 为 `projects` 增加 `status` 字段，支持项目归档状态。
- 二次审查已移除迁移中的 SQLite monkey patch，并回退了对历史迁移 `0001_initial.py`、`0002_partition_strategy.py` 的修改。历史迁移不应为测试适配而被重写。

### 后端能力

- 项目归档：`PATCH /projects/{project_id}` 支持更新项目状态。
- 项目邀请：补齐项目成员邀请持久化链路。
- 导出中心：补齐导出任务、导出产物表和 `GET /exports` 查询接口。
- Provider 测试：允许在无真实密钥或 Sandbox 配置下返回安全降级结果，便于本地和演示环境验证交互链路。
- 配额监控：补齐前端所需的 `/ops/quota` 适配输出。
- 导出历史接口经二次修复后会返回导出 artifacts，避免前端历史列表无法展示下载入口。

### 前端联调

- 导出中心 `/exports` 对接真实导出历史。
- 文档导出流程对接后端任务状态轮询与产物下载。
- 模板上传使用 `FormData` 对接后端上传接口。
- 项目归档从纯前端提示升级为真实 API 调用。
- 前端 API client 支持 JSON 请求与 FormData 请求的不同 header 处理。

## 测试与验证

二次审查后已串行执行以下验证：

```bash
uv run --extra dev python -m pytest
# 263 passed, 5 warnings

pnpm --dir apps/web typecheck
# passed

pnpm --dir apps/web exec playwright test tests/e2e/playwright/deterministic-regression.spec.ts
# 6 passed

pnpm --dir apps/web build
# passed, 22 routes generated

git diff --check
# passed
```

说明：

- 后端测试中的 5 个 RuntimeWarning 为既有测试警告，主要来自 AsyncMock 和 LLM gateway fallback，不是本次改动新增失败。
- 确定性前端 E2E 覆盖全局路由、项目生命周期子路由、文档详情、设置、模板、Providers、Quotas、Agent Operations 和 Workflows。
- `real-api-capability.spec.ts` 为真实 API 联调测试，需要按测试文件要求配置运行环境后启用；本次二次审查没有在当前本地环境额外运行真实 API 测试。

## 二次审查修正

- 回退对 `0001_initial.py`、`0002_partition_strategy.py` 的修改，避免污染已发布历史迁移。
- 移除 `0003_add_v020_tables.py` 中对 `sa.text` 的全局 monkey patch。
- 修复导出列表响应契约，`GET /exports` 返回包含 artifacts 的响应模型。
- 为确定性 E2E 增加 `gotoAppPage` 导航重试，降低 Next.js 路由跳转中 `net::ERR_ABORTED` 造成的偶发失败。
- 修复报告乱码与过度承诺表述，改为可审计的事实记录。

## 剩余边界

- 本分支补齐的是 v0.2.0 当前重点功能所需表和接口，并不代表所有未来业务表都已完全 Alembic 化。
- Provider 的 Sandbox 降级只用于本地、演示或无密钥场景，不等价于真实供应商连通性验证。
- 项目归档、导出、模板上传等能力已具备真实 API 链路，但仍需要在 OCI 环境完成迁移、重建镜像和线上冒烟验证后才算部署完成。
