# AMX 系统说明书与角色操作手册

> 适用系统：Avenir Matrix（AMX）咨询交付 AI 工作台
>
> 编制依据：当前仓库前端页面、API 路由、领域模型、权限实现、CI/CD 与 OCI 运维脚本
> 基准日期：2026-06-14

本目录中的手册描述当前代码已经实现的能力。规划中但尚未形成完整代码闭环的能力，不作为可用功能写入操作步骤。

## 阅读顺序

1. [系统功能综述](./01-system-functional-overview.md)：面向管理者、产品负责人、架构师和实施负责人，说明系统边界、能力地图、核心流程与控制机制。
2. [平台与租户管理员操作手册](./02-platform-tenant-admin-manual.md)：团队、角色、策略、字段权限、API Key、Provider、集成与平台治理。
3. [项目负责人操作手册](./03-project-lead-manual.md)：项目启动、计划、成员、评审、交付、客户验收和项目关闭。
4. [咨询顾问与解决方案架构师操作手册](./04-consultant-architect-manual.md)：资料、知识、对话式文档编写、质量、追溯和智能编排。
5. [业务评审人与审批者操作手册](./05-reviewer-approver-manual.md)：评审、批注、变更、追溯、验收和放行。
6. [外部客户交付门户操作手册](./06-external-customer-manual.md)：使用受控链接下载交付物并提交验收意见。
7. [平台运维与发布负责人操作手册](./07-operations-release-owner-manual.md)：运行监控、故障处置、发布、部署、验证和回滚。

## 重要说明

- 系统采用租户隔离、项目成员关系、RBAC、ABAC 与字段级权限组合授权。页面可见不代表操作一定获准，后端权限判定是最终依据。
- AI 生成内容必须经过人工复核。系统的质量检查、追溯矩阵和交付门禁用于辅助决策，不替代专业责任与正式审批。
- “代码合并”“GitHub Release”“生产 Deployment”是三个不同事件。只有版本发布、生产部署、健康检查和来源校验均成功，才构成完整生产发布。
- 外部客户不进入内部工作台，只能通过有有效期、可撤销的交付门户链接访问限定内容。
