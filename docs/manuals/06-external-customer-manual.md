# 外部客户交付门户操作手册

## 1. 适用范围

本手册面向收到 AMX 客户交付门户链接的外部客户、业务代表或验收负责人。外部客户不需要进入内部工作台，也不能访问内部项目资料、团队权限、运行记录或其他项目。

## 2. 访问交付门户

项目团队会通过安全渠道发送专属链接。该链接具有以下控制：

- 仅访问指定项目的限定交付内容；
- 可以设置有效期；
- 可以由项目团队撤销；
- 系统不保存可直接使用的明文 Token；
- 下载和验收行为形成证据。

访问时建议确认浏览器地址属于正式 AMX 域名。不要转发链接给未授权人员。

若链接过期、被撤销或无法访问，请联系项目负责人重新签发。不要尝试使用其他项目或历史链接。

## 3. 查看交付状态

进入门户后，可查看项目名称、交付包准备状态、可用产物、验收条件和整改状态。

状态说明：

- **可下载/已完成**：产物已完成并允许客户下载。
- **处理中**：产物仍在生成或准备。
- **待处理/整改中**：存在需要项目团队处理的验收事项。
- **已接受**：客户验收意见已被记录为接受。

门户展示的是客户交付范围，不代表内部项目的全部内容。

## 4. 下载交付物

选择允许下载的产物并执行下载。下载后建议：

1. 核对项目名称和文件名称；
2. 确认内容是否完整；
3. 按组织安全要求保存文件；
4. 不将受限交付物提供给未授权人员；
5. 如发现版本或内容不正确，暂停验收并联系项目负责人。

系统会保留下载证据，用于交付追踪。

## 5. 提交验收意见

阅读验收条件和交付物后，按实际结果提交意见。系统支持接受、待处理或拒绝等结果。

### 5.1 接受

当交付物满足约定范围和验收条件时选择接受，并填写必要说明。

### 5.2 待处理

当总体可接受但仍有明确待办时选择待处理。请逐项写明问题、期望结果和优先级，便于项目团队形成整改工作。

### 5.3 拒绝

当存在阻断性问题时选择拒绝。应说明：

- 涉及的交付物；
- 不符合的验收条件；
- 具体问题和影响；
- 可接受的整改结果；
- 期望完成时间。

提交后系统会形成验收记录，并可推动内部整改事项。整改完成后，项目团队可邀请重新验收。

## 6. 验收回执

验收提交成功后，系统记录验收结果和回执。建议保留回执信息，并与合同或组织内部验收流程结合使用。

门户验收记录不自动替代双方合同中约定的签字、盖章、付款或其他法律程序。

## 7. 安全注意事项

- 不向无关人员转发门户链接；
- 不在公开渠道发送下载文件；
- 不使用不匹配的邮箱或身份提交验收；
- 发现链接泄露时立即联系项目负责人撤销；
- 对敏感交付物遵循双方约定的数据保护要求。

## 8. 常见问题

| 问题 | 处理方式 |
|---|---|
| 链接无法打开 | 检查是否过期或被撤销，联系项目负责人 |
| 看不到某个文件 | 文件可能未完成、未列入客户范围或尚未允许下载 |
| 下载文件不正确 | 暂停验收，向项目负责人说明文件和问题 |
| 已提交后需要修改意见 | 联系项目负责人重新打开或发起重新验收 |
| 门户显示整改中 | 等待项目团队处理，并在收到通知后重新验收 |

## 9. 客户验收检查清单

- [ ] 访问的是正式且有效的专属链接
- [ ] 项目和交付范围正确
- [ ] 所需交付物已下载并核对
- [ ] 验收意见具体、准确、可执行
- [ ] 接受、待处理或拒绝选择符合实际
- [ ] 验收回执已保留
