## Rollback Triggers

Rollback immediately if any required production gate fails after deployment:

- `/health` is not healthy after the bounded retry window;
- authenticated smoke cannot log in as the configured bootstrap/admin user;
- current user, projects, documents, readiness, or commissioning smoke fails;
- web dashboard, projects, documents, or settings routes return a client-side exception;
- API or worker service restarts repeatedly;
- migration fails or leaves the API unavailable;
- OCI provenance check reports ref mismatch, SHA mismatch, dirty worktree, or missing required services;
- GitNexus health or index refresh fails when required by the release gate.

## Rollback Procedure

1. Stop promotion activity and preserve the failing GitHub Actions run, health output, smoke output, and OCI provenance output.
2. Re-run the production deployment workflow using the last known good ref, currently `v0.8.8` or SHA `e3808ddaa9519d92d2b4bdd9d238aa7cb177936d`.
3. Wait for the deployment workflow to finish; do not start another production deployment in parallel.
4. Verify `/health`.
5. Run authenticated smoke with the configured production `.env`.
6. Verify OCI provenance:
   - deployed git commit matches the rollback ref;
   - `.deploy-ref` matches the requested rollback ref;
   - working tree is clean;
   - required services are running.
7. Verify GitNexus health and index refresh if the release changed GitNexus-visible code or docs.
8. Record the rollback deployment id, SHA, health result, smoke result, and remaining incident notes in the release issue or GitHub Release notes.

## Database Rollback Policy

Do not run destructive database downgrades automatically.

For v1.0 rollback:

- prefer application rollback to the prior release ref first;
- inspect Alembic migration notes before any downgrade;
- only run a downgrade after confirming no unique customer/project evidence would be lost;
- preserve database backup and migration logs before changing schema state.

## Post-Rollback Checks

Required checks after rollback:

```text
/health
authenticated smoke
OCI provenance
service status
GitNexus health/index refresh when applicable
```

If rollback succeeds, leave the failed `v1.0.0` tag or release unpublished/draft until the owner decides whether to delete or supersede it. Do not rewrite historical release tags without explicit owner approval.
